# Preguntas

## ¿Qué es Attribute-Driven Design (ADD) y cuál es su propósito en el diseño de software?
    el ADD es un metodo que ayuda a diseñar arquitecturas utlizando los atributos de calidad, el 
    proposito de este metodo es que se implemente una arquitectura basado en lo que realmente necesita el sistema.

## ¿Cómo se relaciona ADD con Clean Architecture en el proceso de diseño de sistemas?
    ADD define los atributos de calidad que se deben cumplir en el sistema y la clean arquitecture cumple con estos atributos de forma practica asegurando
    que el software sea mantenible, flexible y escalable.

## ¿Cuáles son los pasos principales del método ADD para definir una arquitectura de software?
    los pasos principales son: 
    1. Identificar los drivers arquitectónicos.
    2. Seleccionar elementos de diseño iniciales.
    3. Definir responsabilidades y módulos.
    4. Crear la arquitectura de alto nivel.
    5. Iterar y refinar en cada nivel.
    
## ¿Cómo se identifican los atributos de calidad en ADD y por qué son importantes?
    se identifican mediante los stakeholder, teneindo en cuenta los requisitos funcionales y entiendo muy bien de lo que trata el sistema.
    Son de importancia porque todo sistema debe asegurar su rendimeintos, calidad, mantenimiento y disponibilidad.
    
## ¿Por qué Clean Architecture complementa ADD en la implementación de una solución?
    porque ADD solo las define en cambio clean architecture tiene todo para cumplirlas como por ejemplo el manejo de capas.
